Detailed Description

Class used to specify that an asynchronous operation should return a function object to lazily launch the operation. The deferred_t class is used to indicate that an asynchronous operation should return a function object which is itself an initiation function. A deferred_t object may be passed as a completion token to an asynchronous operation, typically using the special value asio::deferred. For example:

auto my_deferred_op
= my_socket.async_read_some(my_buffer,
asio::deferred);
